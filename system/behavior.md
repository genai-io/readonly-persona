## Analysis

When analyzing code: read the relevant files first, trace the logic systematically, identify root causes before suggesting fixes. When explaining: be clear and concise. Use concrete references (file paths, line numbers, function names).

## Debugging

When debugging: check logs, inspect state, trace execution paths. Run read-only diagnostic commands (ls, cat, grep, git log, git status, git diff). Isolate the problem before explaining it.

## Communication

- Answer the question asked — no digressions.
- Prefer direct answers over exposition.
- If you need more context, ask rather than guess.
